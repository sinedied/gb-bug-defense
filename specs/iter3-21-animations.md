## 1. Problem

Per the roadmap: *"Animated tiles & sprite polish — Idle animations for towers, hit flashes for
enemies, computer 'corruption' visual as HP drops."* The MVP/iter-2 game looks static between
events: the computer flips between two states (pristine ↔ damaged) at HP≤2, hits on enemies have
no visual feedback, and towers never move. This iteration adds visceral feedback at the
tile/sprite level without adding new gameplay rules, while staying inside the existing
hardware/budget invariants.

### In scope (priority order)
1. **Computer corruption** — replace the binary damaged flip with a 5-state progression keyed to
   exact HP (5→0).
2. **Enemy hit flash** — 3-frame inverted-shading sprite swap on each non-killing projectile hit
   (bug + robot).
3. **Tower idle animation** — subtle 1-pixel LED blink on each placed tower (AV + FW), staggered
   to fit the per-frame BG-write budget.

### Out of scope (deferred per priority/budget)
- **Bug walk frame B** — already shipped in iter-2 (`SPR_BUG_A`/`SPR_BUG_B`, animated every 16
  frames in `enemies.c::step_enemy`). No work.
- **Cursor pulse** — existing `SPR_CURSOR_GA/GB` + 2 Hz blink for invalid placement is solid (see
  cursor convention in `memory/conventions.md`). Iter-3 #22 wired pause-time cursor hiding. No
  work.
- **Title / game-over screen polish** — lowest priority, skipped to stay tight.

## 3. Architecture

### 3.1 Animation timing model — global frame counter

A single `static u8 s_anim_frame` lives in `src/game.c` and is incremented exactly once per
`game_update()` invocation, BEFORE the per-state switch. Exposed as `u8 game_anim_frame(void)`.
Wraps at 256 (1 B). All powers-of-2 phase divisors used by the spec (≥3 = LED period 32, =4 =
stagger spacing 4) divide 256 evenly, so wrap is invisible.

**Rationale**: Global counter is simpler than per-entity counters for the 3 in-scope animations
(none need per-entity phase offsets that aren't derivable from the entity's index). Existing
per-entity anims (enemy `anim` field, cursor `s_blink`) keep their own counters — touching them
risks breaking iter-2 timing.

**Reset boundary**: `enter_playing()` zeros `s_anim_frame` so each session starts at a known phase.
Title and gameover screens still increment it (cheap, no observers).

### 3.2 Computer corruption — HP-keyed 5-state LUT

| HP | state | TL          | TR          | BL          | BR          |
|----|-------|-------------|-------------|-------------|-------------|
| 5  | 0     | `COMP_TL`   | `COMP_TR`   | `COMP_BL`   | `COMP_BR`   |
| 4  | 1     | `COMP_TL_C1`| `COMP_TR`   | `COMP_BL`   | `COMP_BR`   |
| 3  | 2     | `COMP_TL_C2`| `COMP_TR_C2`| `COMP_BL`   | `COMP_BR`   |
| 2  | 3     | `COMP_TL_C3`| `COMP_TR_C3`| `COMP_BL_C3`| `COMP_BR_C3`|
| 1  | 4     | `COMP_TL_D` | `COMP_TR_D` | `COMP_BL_D` | `COMP_BR_D` |
| 0  | 4     | (same as 1; gameover fires the same frame and supersedes any further render) |

Pure mapping function (host-testable, header-only):
```c
static inline u8 map_hp_to_corruption_state(u8 hp) {
    if (hp >= 5) return 0;
    if (hp == 0) return 4;
    return (u8)(5 - hp);   /* 4→1, 3→2, 2→3, 1→4 */
}
```

`map.c` holds a `static const u8 s_comp_tiles[5][4]` LUT and a `static u8 s_state` plus
`s_state_dirty` bit. `map_set_computer_state(u8 hp)` derives the state via the function above and
sets dirty only on change. `map_render()` paints all 4 cluster tiles when dirty (4 BG writes —
worst case once per HP-change event, i.e. ≤ once per 60 frames in practice).

**Session reset**: `map_load()` MUST set `s_state = 0` and `s_state_dirty = false` so a fresh
game starts with the in-memory state matching the freshly-painted pristine cluster (the
`gameplay_tilemap` already encodes state 0). Without this reset, a quit-to-title from a lost run
(`s_state == 4`) → new game would re-trigger 4 wasted BG writes one frame after `enter_playing`.

**Same-frame multi-damage** (e.g. 2 enemies arrive on the same frame and HP 2→1→0):
`economy_damage` fires twice within `enemies_update`; the 2→1 transition dirties state 3→4. Then
`playing_update`'s HP=0 gameover branch fires BEFORE `playing_render` can paint the dirty
cluster — `enter_gameover` does `DISPLAY_OFF` + full-screen redraw and the queued state-4 paint
is dropped. We do NOT add a logic-phase `map_render()` workaround (would violate the
frame-loop split). **Documented limitation**: state 4 is guaranteed visible only when HP
transitions through 1 (single-bug arrival). On simultaneous-multi-damage frames the player goes
straight from state 3 to gameover; this is acceptable because (a) gameover supersedes the
play-field anyway, (b) the iter-2 wave script never spawns simultaneously-arriving enemies in
the bottleneck path. QA scenario 4 is written to cover the single-bug case explicitly.

### 3.3 Hit flash — per-enemy 3-frame timer

```c
typedef struct {
    /* …existing fields… */
    u8 flash_timer;     /* frames remaining in flash sprite override */
} enemy_t;
```

API:
```c
void enemies_set_flash(u8 idx);   /* sets flash_timer = FLASH_FRAMES (3) */
```

`step_enemy()` change (replaces existing tile-select):
```c
u8 tile;
if (enemies_flash_step(&e->flash_timer)) {       /* header-only helper */
    tile = (e->type == ENEMY_BUG) ? SPR_BUG_FLASH : SPR_ROBOT_FLASH;
} else {
    e->anim++;
    u8 frame = (e->anim >> 4) & 1;
    tile = frame ? s_enemy_stats[e->type].spr_b : s_enemy_stats[e->type].spr_a;
}
set_sprite_tile(OAM_ENEMIES_BASE + i, tile);
```

`src/enemies_anim.h` (header-only, self-contained — uses `<stdint.h>` only so
`tests/test_anim.c` can include without pulling `<gb/gb.h>` via `gtypes.h`):
```c
#ifndef GBTD_ENEMIES_ANIM_H
#define GBTD_ENEMIES_ANIM_H
#include <stdint.h>
/* Returns 1 iff a flash is in progress; decrements timer in place. */
static inline uint8_t enemies_flash_step(uint8_t *timer) {
    if (*timer == 0) return 0;
    (*timer)--;
    return 1;
}
#endif
```
On-device callers (`enemies.c`) cast `bool`/`u8` interop is trivial since both alias `u8`.

**Modal freeze**: `step_enemy` is gated by `enemies_update`, which `playing_update` skips while
`game_is_modal_open()`. Therefore `flash_timer` ALSO freezes during pause/menu — consistent with
the "enemies are frozen during modal" iter-2/iter-3 invariant. Do NOT tick `flash_timer` from
`playing_render` or any modal branch (see Decision D16).

`projectiles.c::step_proj` non-killing branch:
```c
if (killed) {
    economy_award(bounty);
    audio_play(SFX_ENEMY_DEATH);
} else {
    enemies_set_flash(p->target);     /* NEW */
    audio_play(SFX_ENEMY_HIT);
}
```

`enemies_init` and `enemies_spawn` set `flash_timer = 0`. `enemies_apply_damage` does NOT touch
the timer (so a killing hit is silent on flash and the OAM hide takes precedence).

`FLASH_FRAMES = 3` is added to `src/tuning.h` so `tests/test_anim.c` can lock it in.

### 3.4 Tower idle animation — round-robin BG-write scanner

LED phase function (header-only, host-testable, in **`src/towers_anim.h`** — separate from
`src/towers.h` because the latter transitively includes `<gb/gb.h>` via `gtypes.h` and
`tests/test_anim.c` must link without GBDK):
```c
#ifndef GBTD_TOWERS_ANIM_H
#define GBTD_TOWERS_ANIM_H
#include <stdint.h>
/* 64-frame period: 32 frames OFF, 32 frames ON. Per-tower phase offset
 * uses a bit-reversed index so adjacent slots (the order placement
 * fills) are spread evenly across the period — slots 0 and 1 are
 * exactly anti-phase (offset 0 vs 32), satisfying QA scenario 10
 * ("place 4 towers in a row → visibly out of phase"). */
static inline uint8_t _towers_bitrev4(uint8_t v) {
    v = (uint8_t)(((v & 0x0Cu) >> 2) | ((v & 0x03u) << 2));
    v = (uint8_t)(((v & 0x0Au) >> 1) | ((v & 0x05u) << 1));
    return (uint8_t)(v & 0x0Fu);
}
static inline uint8_t towers_idle_phase_for(uint8_t frame, uint8_t tower_idx) {
    /* bitrev4 of {0,1,2,…,15} = {0,8,4,12,2,10,6,14,1,9,5,13,3,11,7,15};
     * × 4 = {0,32,16,48,8,40,24,56,4,36,20,52,12,44,28,60} (mod 64).
     * Adjacent placement slots (idx, idx+1) differ by ≥ 16 frames in
     * the 64-frame period (≥ ½ of a 32-frame half-period). */
    uint8_t off = (uint8_t)(_towers_bitrev4(tower_idx & 0x0Fu) << 2);
    return (uint8_t)(((frame + off) >> 5) & 1u);
}
#endif
```
`src/towers.c` includes both `towers.h` (for the public on-device API) AND
`towers_anim.h` (for the inline phase helper).

Per-tower state in `tower_t`:
```c
u8 idle_phase;     /* last-painted LED phase: 0 or 1 */
```

`towers_render()` is restructured (replaces current "1 sell, then 1 place, ≤1 BG write/frame"):

```c
void towers_render(void) {
    /* Phase 1: drain ≤1 sell-clear (unchanged). On hit: return. */
    /* Phase 2: drain ≤1 place (unchanged). On hit: return. */
    /* Phase 3 (NEW): ≤1 idle toggle, gated on !game_is_modal_open(). */
    if (game_is_modal_open()) return;
    u8 cur_frame = game_anim_frame();
    u8 i;
    for (u8 k = 0; k < MAX_TOWERS; k++) {
        i = (s_idle_scan_idx + k) & (MAX_TOWERS - 1);   /* MAX_TOWERS=16 power of 2 */
        if (!s_towers[i].alive) continue;
        u8 want = towers_idle_phase_for(cur_frame, i);
        if (want == s_towers[i].idle_phase) continue;
        u8 base = s_tower_stats[s_towers[i].type].bg_tile;
        u8 alt  = (s_towers[i].type == TOWER_AV) ? TILE_TOWER_B : TILE_TOWER_2_B;
        set_bkg_tile_xy(s_towers[i].tx,
                        s_towers[i].ty + HUD_ROWS,
                        want ? alt : base);
        s_towers[i].idle_phase = want;
        s_idle_scan_idx = (i + 1) & (MAX_TOWERS - 1);
        return;
    }
}
```

`towers_init()` zeros `s_idle_scan_idx` and every tower's `idle_phase`. `towers_try_place()` sets
`idle_phase = 0` (matches the just-painted base tile).

### 3.5 BG-write budget recheck (per-frame, worst-case)

Iter-2 cap: **≤ 16 BG tile writes per VBlank window** (~1080 cycles).

| Source                         | Worst-case writes | Frequency |
|--------------------------------|------------------|-----------|
| HUD digits (HP/E/W/T)          | 10               | rare (digit changes only) |
| Computer corruption swap       | 4                | rare (only on HP change, ≤ once per 60 f) |
| Towers (sell **xor** place **xor** idle) | **1**  | every frame in the worst case (idle keeps it at 1 even when sell/place are idle) |

The restructured `towers_render()` in §3.4 has THREE `return;`-after-write phases (sell → place →
idle), so it emits **at most 1 BG write per frame** total — sell and place are mutually
exclusive across phases by construction; idle scanner runs only when both prior phases were
no-ops.

**Worst single-frame total**: 10 (HUD) + 4 (corruption) + 1 (towers) = **15** ≤ 16. ✅

In the realistic max — typical damage frame: HUD HP digit (1) + corruption (4) + idle (1) =
**6**. Idle scanner steady state: 16 towers × 2 toggles per 64-frame period = 32 events per 64
frames = 0.5 writes/frame average; the 1-write/frame ceiling absorbs bursts. Visual "lag" of
≤16 frames when many towers transition at once is invisible (LED blink half-period = 32 f).

**Sanity guard**: if the idle scanner ever needs > 1 write/frame to keep up, reduce LED period
from 32 → 16 to cut burst size. Not needed at 16 towers; documented as a follow-up if pool
grows.

## 6. Constraints

- **VBlank-safe BG-write budget ≤ 16/frame** — see §3.5; idle scanner is bounded at 1
  write/frame and gated on `!game_is_modal_open()`.
- **Tower idle is gated by modal state** so neither pause nor upgrade/sell menu sees
  surprise BG writes (consistent with iter-3 #22's "no BG writes during modal" stance).

- **OAM allocation** unchanged from iter-3 #22 (`0=cursor, 1..16=menu/pause, 17..30=enemies,
  31..38=projectiles, 39=reserved`).
- **Flash sprite reuses enemy OAM slot** — no new OAM allocation; tile swap only.
- **ROM ≤ 32 KB no-MBC** — projected fill ~16.5 KB / 32 KB. If realised growth > 16 KB
  (total > 32 KB), STOP and file a separate MBC1-conversion spec; do NOT silently expand cart
  type here.
- **Frame loop**: `s_anim_frame++` happens inside `game_update()` (logic phase), NOT in
  `game_render()`. Render phase only reads it.
- **Modal precedence** unchanged. Latch `menu_was_open` invariant from iter-3 #22 untouched.
- **Audio**: no new SFX. Existing `SFX_ENEMY_HIT` already plays on non-killing projectile hits;
  the flash piggybacks on the same call site, no audio reordering.
- **Backwards compat**: removing `map_set_computer_damaged` is a hard rename — there is exactly
  one caller in `src/economy.c`. Verified via `grep -r map_set_computer_damaged src/`.

## 8. Memory Updates

Append to `memory/decisions.md`:

> ### Iter-3 #21: Global anim-frame counter; idle scanner is round-robin, ≤1 BG write/frame, modal-gated
> - **Date**: <fill on commit>
> - **Context**: Iter-3 #21 needs tower idle animation while preserving the iter-2 ≤16 BG writes/frame budget; multiple animation systems need a synchronisation source.
> - **Decision**: Single `static u8 s_anim_frame` in `game.c`, incremented at top of `game_update()`, exposed via `game_anim_frame()`. Tower idle uses `towers_idle_phase_for(frame, tower_idx) = ((frame + (bitrev4(idx & 0xF) << 2)) >> 5) & 1` (see `src/towers_anim.h`) — 32-frame ON / 32-frame OFF; bitrev4 stagger spreads adjacent placement slots across the 64-frame period (slot 0 vs 1 = exactly anti-phase). `towers_render()` Phase 3 round-robin scans for one stale tower and writes ≤1 BG tile/frame; gated on `!game_is_modal_open()`. Reset `s_anim_frame=0` in `enter_playing()`.
> - **Rationale**: Bounds worst-case writes; deterministic phase per session; modal gating consistent with iter-3 #22 "no BG writes during modal" stance.
> - **Alternatives**: Per-entity counters (more state, no benefit for 3 anims); paint all stale towers per frame (busts budget at 16-tower pool); LCD STAT IRQ scheduling (overkill).

> ### Iter-3 #21: Computer state = 5 (HP-keyed, including pristine + 4 corrupted)
> - **Decision**: Replace `map_set_computer_damaged(bool)` with `map_set_computer_state(u8 hp)`. Pure mapping `hp→state` lives in `src/map_anim.h` (header-only, host-testable). State 4 reuses the existing `_D` tile set (HP=1 and HP=0 share a state).
> - **Rationale**: Roadmap explicitly calls for the progressive corruption visual; HP=0 reuse keeps new tile budget at 7 (under the per-iter ROM growth target of <1 KB).

> ### Iter-3 #21: Hit flash = 3-frame sprite override on non-killing hit only
> - **Decision**: Per-enemy `u8 flash_timer`. Set to `FLASH_FRAMES=3` (in `tuning.h`) by `enemies_set_flash()`, called from `projectiles.c::step_proj` when `enemies_apply_damage` returns false. Killing hits do NOT flash (sprite hides immediately).
> - **Rationale**: Matches the "feedback on partial damage" intent; killing-hit hide is louder than any flash.

Append to `memory/conventions.md`:

> ### Iter-3 #21 conventions
> - **Global anim phase**: `game_anim_frame()` (in `game.h`) is the single source of truth for cross-module animation timing. Wraps at 256; all phase divisors used (32 = LED half-period; bitrev4 stagger across 64-frame full period) divide it evenly. Reset to 0 in `enter_playing()`.
> - **BG-write budget (revised)**: ≤ 16 writes/frame remains the cap. `towers_render` still contributes **at most 1** write per frame total — sell, place, and idle phases each `return;` after writing, so they are mutually exclusive within a single frame. Updated worst-case: 10 (HUD) + 4 (corruption) + 1 (towers) = **15**. The idle scanner's 1-write/frame ceiling is non-negotiable; do not "drain all stale" in one frame.
> - **Modal gating extended**: any new "ambient" BG-write source MUST gate on `!game_is_modal_open()` (towers idle is the precedent). Animation timers tied to entity ticks (e.g. `flash_timer` inside `enemies_update`) freeze automatically because their host `_update` is already gated. Do NOT add cross-module tick paths that bypass modal gating.
> - **Pure-helper headers**: `src/map_anim.h`, `src/enemies_anim.h`, and `src/towers_anim.h` join `src/tuning.h` and `src/game_modal.h` as host-testable header-only modules. All three new headers use `<stdint.h>` directly (NOT `gtypes.h`, which transitively pulls `<gb/gb.h>`). Tests live in `tests/test_anim.c` and link without GBDK stubs.
